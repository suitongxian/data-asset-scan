file_statistics增加了三个字段
 - 工作空间文件认领数：
 - 历史文件认领数
 - 非历史文件认领数


查询data_resources 表实现一个统计接口 
- 总文件数：所有记录的 sum(source_count)
- 工作空间总文件数：所有记录的 sum(workspace_source_count)
- 历吏文件总数：first_create_time < 历吏封帐时间 记录的 sum(source_count)
- 非历吏文件总数：first_create_time > 历吏封帐时间 记录的 sum(source_count)
- 工作空间文件认领数：claim_status > 0 记录的 sum(workspace_source_count)
- 历史文件认领数： first_create_time < 历吏封帐时间 并且 claim_status > 0 记录的 sum(source_count)
- 非历史文件认领数： first_create_time > 历吏封帐时间 并且 claim_status >0 记录的 sum(source_count)
- 未分类文件数：importance_level=1 的 sum(source_count)
- 核心文件数：importance_level=1 的 sum(source_count)
- 重要文件数：importance_level=2 的sum(source_count)
- 开放文件数：importance_level=3 的sum(source_count)
- 隐私文件数: importance_level=4 的 sum(source_count)

- 工作空间待归类保护数 : 满足以下条件的记录数 claim_status==2  && importance_level==0 && workspace_source_count >0
- 历吏文件待归类保护数 : 满足以下条件的记录数 claim_status==2  && importance_level==0 && first_create_time < 历吏封帐时间
- 非历吏文件待归类保护数: 满足以下条件的记录数 claim_status==2  && importance_level==0 && first_create_time > 历吏封帐时间

历吏封帐时间 在system_config表的中 FULL_INVENTORY_TIME 值 ，如果没有该值 与之前相关的统计字段结果置为 -1
